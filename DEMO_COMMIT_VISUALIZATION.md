# 🎯 提交可视化功能演示

这是一个演示文件，用于展示我们新实现的提交可视化功能。

## ✨ 功能亮点

### 自动化分析
- 📊 **智能统计**: 自动分析每次提交的代码变更量
- 🏷️ **文件分类**: 根据文件类型进行智能分类和标识
- 📈 **趋势展示**: 可视化展示代码变更趋势

### GitHub集成
- 🤖 **自动触发**: 推送代码时自动生成分析报告
- 💬 **PR评论**: 在Pull Request中自动添加变更摘要
- 📦 **报告存储**: 生成的报告作为GitHub Actions产物保存

### 本地支持
- 🖥️ **本地生成**: 支持在本地环境生成可视化报告
- 🎨 **美观界面**: 生成专业的HTML可视化报告
- ⚡ **快速预览**: 一键生成Markdown格式摘要

## 🚀 使用示例

### 在GitHub上
当你推送代码到仓库时，GitHub Actions会自动：
1. 分析代码变更
2. 生成可视化报告
3. 在PR中添加摘要评论

### 在本地
```bash
# 生成当前提交的可视化报告
npm run commit-summary

# 查看生成的报告
open commit-reports/commit-report-[hash].html
```

## 📊 报告内容

生成的报告包含：
- 📁 变更文件统计
- ➕ 新增代码行数
- ➖ 删除代码行数
- 🏷️ 文件类型分类
- 📝 详细变更列表

## 🎨 可视化效果

报告采用现代化设计，包含：
- 🎯 统计卡片展示
- 🌈 颜色编码分类
- 📱 响应式布局
- ✨ 交互式界面
